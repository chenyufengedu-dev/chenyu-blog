## 为什么选择 MDX？

纯粹的 Markdown 只能排版静态文本，而 MDX 允许我在文章中直接注入 React 组件。这意味着未来如果我想在一篇讲解算法的文章里插入一个可以交互的 Canvas 动画，或者一个数据可视化的图表，都可以无缝实现。

保持极简，保持克制。

```python
    <header className="mb-12 border-b border-border pb-8">
        <h1 className="mb-6 text-3xl font-semibold tracking-tight text-text-primary sm:text-4xl leading-[1.2]">
          {metadata.title}
        </h1>
        <div className="flex flex-wrap items-center gap-4 text-sm text-text-muted">
          <time dateTime={metadata.date} className="font-mono">
            {metadata.date}
          </time>
          <div className="h-1 w-1 rounded-full bg-border"></div>
          <div className="flex gap-3">
            {metadata.tags.map((tag) => (
              <span key={tag} className="text-accent">
                #{tag}
              </span>
            ))}
          </div>
        </div>
      </header>
```
